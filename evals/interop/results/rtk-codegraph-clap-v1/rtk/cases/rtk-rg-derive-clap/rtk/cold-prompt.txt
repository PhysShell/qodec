The payload is encoded as a `%q1` container. Format: first line
`%q1 <codec> ...` (parameters), then legend lines `<alias>=<phrase>`
(each alias is a short stand-in for that exact phrase), then a
`%q1 body` line, then the body. `%q1 xN` after a line means that line
occurs N times in total. A `toon` body is a table: first line is a JSON
array of keys, each following line is one object, cells are JSON values
joined by the separator named in the header. A `grep` body is grouped
matcher output: a marker line names a file path and the lines after it
are `line:text` hits in that file (a bare marker starts verbatim
lines). A `diag` body line starting with an alias expands to: the head
after the alias, then that alias's legend template with each slot
placeholder filled by the sep-joined values in order. A `tmpl` body
line starting with an alias is its legend template with the slot
placeholders filled by the sep-joined values that follow the alias.
Mentally decode the body via the legend before answering; never emit
alias characters in answers.
